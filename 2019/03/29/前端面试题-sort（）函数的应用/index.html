前端面试题-sort（）函数的应用

题目

使用 sort() 对数组 [3, 15, 8, 29, 102, 22] 进行排序，输出结果
直接使用调用Array的sort()函数

看到这个结果，一定会有人说，what the * ? 怎么回事，2怎么会会跑到中间了？这种情况和sort函数的内部实现有关，sort排序的时候会先调用每个数组项的toString（）方法，然后比较字符串的大小，而对比字符串的时候呢，对于数字来说，它先对比第一位，所以第一位为1的数字都排前面了，而8则排到最后，而第一位相同的时候就比较第二位，第二位大的则排在后面，所以15排在102后面，就是因为上面的原因，所以才会出现上面那个奇奇怪怪的结果。
备注：上面的比较是根据unicode位点进行比较的

上面代码传入一个比较函数compare，它接受两个参数，a，b。当返回a-b时，排序的顺序是升序，反之则是降序。原因如下：

  • 如果 compare(a, b) 小于 0 ，那么 a 会被排列到 b 之前；
  • 如果 compare(a, b) 等于 0 ， a 和 b 的相对位置不变。备注： ECMAScript 标准并不保证这一行为，而且也不是所有浏览器都会遵守（例如 Mozilla 在 2003 年之前的版本）；
  • 如果 compare(a, b) 大于 0 ， b 会被排列到 a 之前。
  • compare(a, b) 必须总是对相同的输入返回相同的比较结果，否则排序的结果将是不确定的。
